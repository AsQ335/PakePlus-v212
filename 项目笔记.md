# 打包设置

您询问的是 `ppconfig.json` 文件中 `desktop` 配置对象的各个属性含义。这是 PackPlus 项目用于桌面端应用打包的配置文件 PakePlus:149-168行 。

## 配置属性说明

### 基础信息
- **`name`**: 应用的英文名称,不支持空格 PakePlus:30-30 
- **`showName`**: 应用的显示名称,支持中文 PakePlus:31-31 
- **`version`**: 应用版本号,用于区分不同版本 PakePlus:32-32 
- **`id`**: 应用的唯一标识符,不能与其他项目重复 PakePlus:33-33 
- **`desc`**: 项目描述信息 PakePlus:34-34 

### URL 和资源路径
- **`webUrl`**: 要打包的网站地址或本地 HTML 文件路径 PakePlus:35-35 
- **`iconPath`**: 应用图标的输入路径 PakePlus:36-36 
- **`inputPath`**: 图标处理的输入路径 PakePlus:37-37 
- **`tempPath`**: 处理后图标的临时存储路径 PakePlus:38-38 
- **`icnsPath`**: macOS 图标文件(.icns)的输出路径 PakePlus:39-39 

### 发布信息
- **`pubBody`**: 发布时的内容描述 PakePlus:40-40 

### 功能开关
- **`isHtml`**: 是否为本地 HTML 项目 PakePlus:41-41 
- **`single`**: 是否启用单例模式,防止多个应用实例同时运行 PakePlus:42-42 
- **`state`**: 是否保持窗口状态(大小和位置),重启后恢复 PakePlus:43-43 
- **`injectJq`**: 是否注入 jQuery 库 PakePlus:44-44 
- **`tauriApi`**: 是否启用 Tauri API,允许从 JavaScript 调用 Tauri 接口 PakePlus:45-45 
- **`debug`**: 是否开启调试模式 PakePlus:46-46 

### 构建方式
- **`buildMethod`**: 构建方法,值为 "local" 表示本地构建 PakePlus:166-166 

## 备注
这些配置项在 `scripts/ppconfig.json` 中定义 PakePlus:149-168 ,用于自定义打包桌面应用 PakePlus:26-27 。如果需要修改移动端配置,应修改 `android` 或 `ios` 字段 PakePlus:27-27 。
